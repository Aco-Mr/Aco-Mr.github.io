MySQL主从同步实战

Windows10启动多MySQL服务

1、将安装的MySQL复制一份

2、修改MySQL的my.ini文件

[mysqld]
# 设置3307端口
port=3307
# 安装mysql的路径
basedir=E:\Program Files\MySQL\MySQL3307
# mysql的数据存放目录
datadir=E:\ProgramData\MySQL\MySQL3307\data
# 允许最大连接数
max_connections=200
# 允许连接失败的次数。
max_connect_errors=10
character-set-server=utf8
# 默认存储引擎
default-storage-engine=INNODB
#mysql_native_password
default_authentication_plugin=mysql_native_password
[mysql]
# 设置mysql客户端默认字符集
default-character-set=utf8
[client]
port=3307
default-character-set=utf8

注意：若安装时是自定义路径，得在自定义的路径下找到ProgramData文件才能找到my.ini和data文件

3、删除data文件

4、注册服务

(1) 初始化复制的MySQL

进入MySQL3307的bin目录下打开cmd

mysqld install MySQL3307 --defaults-file="E:\ProgramData\MySQL\MySQL3307\my.ini"

安装成功

(2) 初始化数据库

在MySQL3307的bin目录输入

mysqld --initialize

(3) 初始化数据库

初始化完成之后MySQL3307的data目录下就会生成一个data文件，找到.err文件打开，这就是生成好的临时密码

(4) 启动数据库

启动命令

net start MySQL3307

5、进入数据库

(1) 登录

在MySQL的bin目录下进入cmd，再将临时密码输入

mysql -P3307 -uroot -p

(2) 修改密码

ALTER USER 'root'@'localhost' IDENTIFIED BY '新密码';

Mysql实现主从复制

1、修改主数据库的my.ini配置文件

# **** 主从同步配置 ****
# 设置优先级，主必须小于从，必须唯一
server-id=1
# bin log文件前缀
log-bin="SC-201910302109-bin"
binlog_do_db=jdbc    #待同步的数据库日志
# 不需要同步的数据库
binlog_ignore_db=information_schema
binlog_ignore_db=mysql
binlog_ignore_db=performance_schema
binlog_ignore_db=test
# 设置 binlog日志格式 STATEMENT, ROW, MIXED
binlog_format=mixed
# 设置一个 binlog 文件的最大字节
# 设置最大 100MB
max_binlog_size=104857600
# 设置了 binlog 文件的有效期（单位：天）
expire_logs_days = 7

修改后启动或重启主数据库

2、修改从数据库的my.ini配置文件

# **** 主从同步配置 ****
server-id=202    #服务器 id，随意，但要唯一
log-bin=mysql-bin   #二进制文件存放路径
# 只读模式
read_only=1
# 需要同步的数据库
replicate_do_db=jdbc

# 设定需要复制的表
# replicate_do_table=jdbc.tab1
# 设定需要忽略的复制表
# replicate_ignore_table=jdbc.tab2

# 增加通配符的两个配置
# replicate_wild_do_table=jdbc.%   只复制哪个库的哪个表
# replicate_wild_ignore_table=jdbc.%   忽略哪个库的哪个表

# replicate_do_table 跟 replicate_wild_do_table 一样，只不过replicate_wild_do_table可以加通配符
# replicate_ignore_table 跟 replicate_wild_ignore_table 一样，只不过replicate_wild_ignore_table可以加通配符

#不同步的数据库
binlog-ignore-db=mysql
binlog-ignore-db=information_schema
binlog-ignore-db=performance_schema  

# 不需要同步的数据库
replicate_ignore_db=information_schema
replicate_ignore_db=mysql
replicate_ignore_db=performance_schema

binlog_cache_size=1M

##主从复制的混合模式
#binlog_format=mixed
#忽略主键重复的错误
#slave_skip_errors=1062

修改后启动或重启从数据库

3、配置主从数据库同步

注意：MySQL提供的主从同步机制中，从库是读取主库的binlog日志进行同步操作的，因此，主库开启同步之前的数据从库是无法读取的，只能把主库的数据dump下来进行人工对从库进行数据同步操作。

(1) 主库创建同步账户并授权

-- 方式一
-- 创建用户
CREATE USER 'aco'@'%' IDENTIFIED WITH mysql_native_password BY '123';
-- 授权
GRANT REPLICATION SLAVE ON *.* TO 'aco'@'%';
-- 刷新权限
FLUSH PRIVILEGES

-- 方式二
-- 创建并授权
grant replication slave,replication client on *.* to 'aco'@'127.0.0.1' identified by '123456';
-- 刷新权限
FLUSH PRIVILEGES

(2) 上读锁，进行锁库操作，防止同步时主库被写入数据

-- 锁库，只能读
FLUSH TABLES WITH READ LOCK

(3) 查看主库信息并记录相关字段

-- 查看状态
show master status;

-- 查询结果
| File	                      | Position
| SC-201910302109-bin.000043	| 1026

(4) 从库开启同步命令

-- 关闭从库
stop slave;
-- 设置同步，注意这里是主库ip，日志名称和位置是我们之前上图中看到的名称和位置
change master to 
master_host='127.0.0.1', -- 主库ip地址
master_user='aco', -- 主库的用户账号
master_password='123456', -- 主库的用户密码
master_port=3306, -- 主库的端口
master_log_file='SC-201910302109-bin.000043', -- 查询主库状态信息的File字段
master_log_pos=1026; -- 查询主库状态信息的Position字段
-- 重置从库
reset slave
-- 开启从库
start slave; 
-- 检查服务器状态
show slave status

-- 查询结果这两个字段都为yes则开启成功
| Slave_IO_Running |	Slave_SQL_Running
| Yes	             |  Yes

(5) 恢复主库读锁状态

-- 解锁
UNLOCK TABLES

4、踩坑

(1) 报错ERROR 1819 (HY000): Your password does not satisfy the current policy requirements

- 原因

密码设置的太简单，不符合mysql密码设置规范

- 解决方法

1、查看mysql完整的密码规则
show variables like ‘validate_password%’;
注意：若显示为Empty set, 1 warning(0.00 sec) 则需要安装validate_password插件。

/** 安装插件 **/
mysql> INSTALL COMPONENT 'file://component_validate_password';
Query OK, 0 rows affected (0.10 sec)

mysql> show variables like 'validate_password%';
+--------------------------------------+--------+
| Variable_name                        | Value  |
+--------------------------------------+--------+
| validate_password.check_user_name    | ON     |
| validate_password.dictionary_file    |        |
| validate_password.length             | 8      |
| validate_password.mixed_case_count   | 1      |
| validate_password.number_count       | 1      |
| validate_password.policy             | MEDIUM |
| validate_password.special_char_count | 1      |
+--------------------------------------+--------+
7 rows in set, 1 warning (0.00 sec)

2、更改密码长度
set global validate_password_length=4;

3、设置密码验证策略(medium【中】,low【低】)
set global validate_password_policy=0;

(2) 主从同步的加密方式不一致问题

报错信息:error connecting to master ‘mysql@192.168.28.3:3306’ - retry-time: 60 retries: 1 message: Authentication plugin ‘caching_sha2_password’ reported error: Authentication requires secure connection.

- 原因

加密方式和解密方式不一致，需要修改密码加密规则

- 解决方法

修改主库账号密码规则

-- 修改用户的密码策略
ALTER USER 'aco'@'%' IDENTIFIED WITH mysql_native_password BY '123456';

MySQL实现读写分离

1、读库创建账户

-- 创建用户
CREATE USER 'aco_slave'@'%' IDENTIFIED WITH mysql_native_password BY '123';

2、授权用户只读权限

-- 授权
GRANT SELECT ON *.* TO 'aco_slave'@'%';

3、刷新权限

-- 刷新权限
FLUSH PRIVILEGES
